# ГРАФЪ — Telegram Mini App барбершопа

Учебный демо-проект: стильное мини-приложение барбершопа с ИИ-консультантом
и пошаговой записью. Стек: **React 18 + Vite**, без базы данных (всё состояние
в памяти приложения).

## Структура проекта

```
grafb-project/
├─ index.html              # точка входа, подключает Telegram WebApp SDK
├─ package.json            # зависимости и команды
├─ vite.config.js          # конфиг сборщика (base: "./")
├─ .env.example            # пример переменных окружения
├─ .gitignore
├─ src/
│  ├─ main.jsx             # монтирование React
│  ├─ App.jsx              # ВСЁ приложение: данные, экраны, стили, Telegram SDK
│  └─ assets/              # фотографии (мастера, hero, галерея) — 14 файлов
└─ server/
   └─ proxy.mjs            # пример бэкенда-прокси для ИИ (чтобы не светить ключ)
```

Почти весь код — в `src/App.jsx`. Сверху файла лежат данные, которые удобно
менять: `SALON` (контакты), `SERVICES` (услуги и цены), `BARBERS` (мастера),
`GALLERY`, тексты. Фото мастеров — это импорты из `src/assets/`.

## Быстрый старт

```bash
npm install
npm run dev
```

Откроется на `http://localhost:5173`. ИИ-чат сразу работает в **демо-режиме**
(простая заглушка по ключевым словам) — настоящий бэкенд для этого не нужен.

Сборка для продакшна:

```bash
npm run build      # результат в папке dist/
npm run preview    # локальный предпросмотр собранной версии
```

## ИИ-консультант: демо-режим vs настоящий ИИ

По умолчанию чат отвечает заглушкой (`mockConsultant` в `App.jsx`) — этого
достаточно, чтобы показать сценарий «спросил → предложили записаться».

Чтобы подключить настоящую модель (`gpt-4o-mini`), нужен **бэкенд-прокси**
(ключ OpenAI нельзя держать во фронтенде — его украдут из исходников):

1. Поднимите прокси из `server/proxy.mjs`:
   ```bash
   npm install express cors
   OPENAI_API_KEY=sk-... node server/proxy.mjs
   ```
2. Скопируйте `.env.example` в `.env` и впишите адрес прокси:
   ```
   VITE_AI_ENDPOINT=http://localhost:8787/api/chat
   ```
3. Перезапустите `npm run dev`. Теперь чат ходит в настоящую модель.

Ключ задаётся на сервере через переменную `OPENAI_API_KEY` и **никогда** не
попадает во фронтенд-бандл.

## Деплой на VPS (Timeweb)

Сервер `server/proxy.mjs` умеет и проксировать запросы к OpenAI (`/api/chat`),
и отдавать собранный сайт из `dist/` — поэтому на VPS всё работает **одним
процессом Node на одном порту**, без CORS. Адрес API относительный
(`/api/chat`, прописан в `.env.production`), так что фронтенд и бэкенд всегда
на одном домене.

Нужен Node.js 18+ (для встроенного `fetch`).

```bash
# на сервере, в каталоге проекта:
npm install                 # ставит фронтенд-зависимости + express/cors/dotenv
npm run build               # собирает фронтенд в dist/ (берёт VITE_AI_ENDPOINT=/api/chat)
cp .env.example .env        # затем впишите в .env свой OPENAI_API_KEY
nano .env
npm start                   # поднимает сервер на :8787 (отдаёт сайт + /api/chat)
```

Ключ хранится в `.env` (он в `.gitignore`); сервер читает его сам через
`dotenv`. В команде запуска ключ указывать не нужно.

**ProxyAPI (доступ к OpenAI из РФ).** Сервис OpenAI-совместимый, поэтому
достаточно в `.env` указать ключ ProxyAPI и базовый URL:

```ini
OPENAI_API_KEY=<ключ ProxyAPI>
OPENAI_BASE_URL=https://api.proxyapi.ru/openai/v1
```

Авторизация и формат запросов те же (`Bearer`-заголовок). Если оставить
`OPENAI_BASE_URL` пустым — сервер пойдёт напрямую в `https://api.openai.com/v1`.

Чтобы процесс не падал и поднимался после перезагрузки — запускайте под
менеджером процессов. В репозитории уже есть готовые конфиги.

**Вариант А — pm2** (`ecosystem.config.cjs`):

```bash
npm install -g pm2
pm2 start ecosystem.config.cjs
pm2 save && pm2 startup
```

**Вариант Б — systemd** (`deploy/grafb.service`): отредактируйте пути в файле
под свой сервер и следуйте инструкции в его шапке (`sudo cp … && systemctl
enable --now grafb`).

Для HTTPS и нормального домена поставьте перед Node **nginx** как
reverse-proxy (Mini App работает только по HTTPS):

```nginx
server {
    server_name grafb.example.ru;
    location / {
        proxy_pass http://127.0.0.1:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Сертификат — `certbot --nginx`. После этого сайт доступен по HTTPS, а
`/api/chat` проксируется в Node автоматически (отдельный конфиг не нужен).

> Можно и без Node на отдаче статики: тогда nginx раздаёт `dist/` напрямую,
> а в `location /api/` проксирует на `http://127.0.0.1:8787`. Относительный
> `/api/chat` работает в обоих вариантах.

## Деплой как Telegram Mini App

1. Разверните сайт на VPS по HTTPS (см. раздел «Деплой на VPS» выше) —
   это даёт и сайт, и работающий ИИ-чат на одном домене. Mini App работает
   **только по HTTPS**.
   _(Если ИИ-чат не нужен, можно залить только `dist/` на статик-хостинг —
   GitHub Pages, Netlify, Cloudflare Pages — чат останется в демо-режиме.)_
2. В Telegram у [@BotFather](https://t.me/BotFather):
   - создайте бота (`/newbot`), если ещё нет;
   - привяжите URL приложения через `/newapp` либо задайте кнопку меню
     командой `/setmenubutton` и укажите адрес вашего сайта.
3. Откройте бота в Telegram — приложение запустится во встроенном webview.

После запуска внутри Telegram автоматически активируются:
- нативная кнопка «Назад» (`BackButton`);
- нативная главная кнопка (`MainButton`) — «Записаться» / «Подтвердить»;
- тактильная отдача (`HapticFeedback`) на выборах и подтверждении;
- тёмный цвет шапки и фона под тему приложения.

Вне Telegram (в обычном браузере) всё это просто не активируется — сайт
работает как обычный одностраничник.

## Важно про учебный характер

Записи никуда не сохраняются — после подтверждения показывается экран
с «кодом брони», но это лишь визуализация. Нет базы данных, нет админки,
нет реальных уведомлений мастерам. Для боевого использования понадобятся
бэкенд с хранением записей и интеграция с расписанием мастеров.
